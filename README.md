# openfoam-cfd-minilab

Small OpenFOAM CFD mini-lab for a traditional incompressible-flow workflow. The included case is a 2D laminar lid-driven cavity at Reynolds number 100.

## Current Status

Case setup is provided and validated by repository tests. In this Codex environment, `blockMesh`, `checkMesh`, and `icoFoam` are not available on `PATH`, so OpenFOAM was not executed locally here.

The local `results/` and `figures/` directories now contain a downloaded GitHub Actions artifact from an OpenFOAM 11 run started at `2026-06-14 16:55:14 UTC` according to `results/logs/workflow_debug.log`. These are executed CI solver results, not outputs fabricated in the local Codex environment.

Artifact evidence currently present:

- `results/logs/blockMesh.log`: `blockMesh` wrote a `40 x 40 x 1` mesh with 1600 cells.
- `results/logs/checkMesh.log`: `checkMesh` reports `Mesh OK`.
- `results/logs/icoFoam.log`: `icoFoam` reaches `Time = 5s` and ends with `End`.
- `results/residuals.csv`: parsed residual history with 4000 data rows.
- `results/centerline_u.csv` and `results/centerline_v.csv`: nearest-cell centerline profiles with 40 data rows each.
- `figures/cavity_residuals.png`, `figures/cavity_centerline_profiles.png`, and `figures/cavity_velocity_magnitude.png`: non-empty PNG figures generated from the CI outputs.

## Governing Equations

The case uses the incompressible laminar Navier-Stokes equations:

```text
div(U) = 0
dU/dt + div(U U) = -grad(p) + nu laplacian(U)
```

where `U` is velocity, `p` is kinematic pressure, and `nu` is kinematic viscosity.

## Physical Setup

- Geometry: 2D unit square cavity, `0 <= x <= 1`, `0 <= y <= 1`
- Effective 2D treatment: thin `z` direction with `empty` front/back patches
- Lid velocity: `U = (1, 0, 0)`
- Characteristic length: `L = 1`
- Kinematic viscosity: `nu = 0.01`
- Reynolds number: `Re = U L / nu = 100`
- Solver: `icoFoam`, suitable for transient incompressible laminar flow

The GitHub Actions workflow targets OpenFOAM Foundation v11 using the `openfoam/openfoam11-paraview510:11` Docker image. For that solver environment, the tracked viscosity file is `cases/lid_driven_cavity/constant/physicalProperties`, which sets `nu = 0.01`. The older `constant/transportProperties` file is retained for compatibility with OpenFOAM variants that still read that dictionary.

## Mesh

The default case uses a `40 x 40 x 1` structured mesh generated by `blockMesh`. A coarser `20 x 20 x 1` mesh is also included. The run script selects from:

- `cases/lid_driven_cavity/system/blockMeshDict.20x20`
- `cases/lid_driven_cavity/system/blockMeshDict.40x40`

and copies the selected dictionary to `cases/lid_driven_cavity/system/blockMeshDict` before `blockMesh`.

Generated mesh files under `cases/lid_driven_cavity/constant/polyMesh/` are intentionally ignored by git and can be regenerated.

## Boundary Conditions

Velocity:

- `top`: moving wall, `movingWallVelocity`, value `(1 0 0)`
- `bottom`, `left`, `right`: no-slip walls
- `frontAndBack`: `empty`

Pressure:

- Walls: `zeroGradient`
- `frontAndBack`: `empty`
- Pressure reference: `pRefCell 0`, `pRefValue 0` in `fvSolution`

## Reproduction Commands

Run from the repository root inside an OpenFOAM-enabled shell:

```bash
bash scripts/run_cavity.sh
```

Run the included mesh refinements explicitly:

```bash
MESH_RESOLUTION=20 bash scripts/run_cavity.sh
MESH_RESOLUTION=40 bash scripts/run_cavity.sh
```

The script runs:

```bash
cp system/blockMeshDict.${MESH_RESOLUTION:-40}x${MESH_RESOLUTION:-40} system/blockMeshDict
blockMesh
checkMesh
icoFoam
postProcess -case cases/lid_driven_cavity -func writeCellCentres -latestTime   # optional, best effort
foamToVTK -case cases/lid_driven_cavity -latestTime -fields "(U)"              # optional, best effort
python scripts/plot_residuals.py --log results/logs/icoFoam.log --output figures/cavity_residuals.png --csv results/residuals.csv
python scripts/postprocess_cavity.py --case cases/lid_driven_cavity --results results --figures figures
```

Clean generated OpenFOAM and post-processing outputs:

```bash
bash scripts/clean_case.sh
```

Run repository tests:

```bash
python -m pytest
```

## Cloud reproduction with GitHub Actions

Local OpenFOAM commands were unavailable in the Codex environment used to create this scaffold. The executed solver results currently present under `results/` and `figures/` came from the GitHub Actions workflow artifact, not a local OpenFOAM run.

The manual GitHub Actions workflow at `.github/workflows/reproduce.yml` runs the required OpenFOAM solver reproduction, `blockMesh/checkMesh/icoFoam`, in the `openfoam/openfoam11-paraview510:11` OpenFOAM Docker image. This OpenFOAM 11 workflow uses `constant/physicalProperties` for the kinematic viscosity. It then runs Python plotting and output validation on the Actions host with `actions/setup-python`, so the Docker container only needs to provide the OpenFOAM tools.

To run it:

1. Push this repository to GitHub.
2. Open the repository's **Actions** tab.
3. Select **Reproduce OpenFOAM Lid-Driven Cavity**.
4. Choose **Run workflow**.
5. After the workflow finishes, download the artifact named `openfoam-cavity-results`.

The workflow distinguishes case setup from executed solver results:

- Case setup is the tracked OpenFOAM dictionaries and scripts in this repository.
- Executed solver results are the logs, derived CSVs, and figures generated by the workflow after `blockMesh`, `checkMesh`, `icoFoam`, and Python post-processing run.

OpenFOAM 11 in the CI image does not rely on the legacy `sample` command for required outputs. Centerline profiles are generated by Python from final-time OpenFOAM field output: the final `U` volVectorField and either a `C` cell-centre field from best-effort `writeCellCentres` or reconstructed structured cell centres from `system/blockMeshDict`. The extraction uses nearest-cell values along `x = 0.5` and `y = 0.5`.

The workflow fails if required outputs are missing or empty. It uploads `results/` and `figures/` as the `openfoam-cavity-results` artifact for inspection. The current local artifact includes the required solver logs, residual CSV, centerline CSVs, residual plot, centerline plot, and velocity-magnitude figure.

## Limitations

- OpenFOAM was not executed in the local Codex environment; the present solver outputs were generated by GitHub Actions and copied into this working tree.
- The default mesh is a modest `40 x 40` teaching mesh, not a grid-convergence study.
- The centerline profiles use nearest-cell extraction from final-time cell-centred values, not benchmark interpolation or validation data.
- ParaView remains the recommended local tool for detailed field inspection.
